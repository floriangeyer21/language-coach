# Feature: Chat with AI

**Status:** planned (in `new/`)

## Goal

A contextual conversation with an AI language coach. The coach knows who the user is (via memory), teaches in the user's target `learning_language`, and can act on the user's data (e.g. save vocabulary) during the conversation.

## Scope (v1)

- Single ongoing conversation thread per user.
- Send a message → receive an assistant reply generated by OpenAI (`OPENAI_CHAT_MODEL`).
- Reply is grounded in the user's memory (facts + summary + last 10 messages) — see [[memory-management]].
- Streaming responses (token-by-token) preferred; non-streaming supported.
- The AI can add vocabulary words on the user's behalf via a tool call (see below).
- View full conversation history; clear the conversation.

## AI Behavior

- System prompt establishes: role = patient language coach; the user's `learning_language`; that the user's native language is English; a concise, encouraging teaching style.
- The model is provided with function-calling tools and may invoke them mid-turn.

### Tools

- **`add_vocabulary_word`** `{ term, translation?, group_name? }` — saves a word to the user's vocabulary. If `group_name` omitted, uses the default group. Enables flows like: user says "add *Apfel* to my food words" → AI calls the tool → word saved. Executed side effects are returned in the response `actions` array so the UI can confirm them. See [[vocabulary]].

## Turn Lifecycle

1. Persist user message.
2. Assemble prompt from memory tiers (see [[memory-management]]).
3. Call OpenAI (optionally streaming); handle any tool calls, executing them against the user's data.
4. Persist + return assistant message and `actions`.
5. Trigger async memory update (does not block response).

## Data (conceptual)

- `message`: `id`, `user_id`, `role` (`user`|`assistant`), `content`, `created_at`.

## Out of scope (v1)

Multiple/named conversation threads, voice input/output, image messages, non-OpenAI providers (pluggable later).
